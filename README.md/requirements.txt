# TransactionETLDB ETL & Data Analytics Project

## Project Overview

This project is a complete end-to-end ETL (Extract, Transform, Load) and data analytics pipeline built using a transaction dataset. The project focuses on extracting raw transaction data, transforming and cleaning the data using Python and SQL Server, and generating business insights through SQL analysis and Power BI visualizations.

The final solution demonstrates how transactional data can be processed into a structured analytical dataset to support business intelligence, fraud monitoring, revenue analysis, and operational reporting.

---

## Project Objectives

- Build a professional ETL pipeline
- Clean and transform raw transaction data
- Load processed data into SQL Server
- Perform SQL-based business analysis
- Create interactive Power BI dashboards
- Publish a professional GitHub portfolio project

## ETL Pipeline

### 1. Extract

- Imported raw transaction CSV data
- Loaded dataset into Python using Pandas
- Validated data structure and columns

### 2. Transform

Data cleaning included:

- Removing duplicate records
- Handling null values
- Formatting transaction amounts
- Standardizing transaction types
- Cleaning fraud flag values
- Validating timestamps

### 3. Load

- Loaded transformed data into SQL Server
- Created clean analytical tables
- Validated total row counts

## Sample SQL Analysis

### Transaction Revenue by Type

```sql
SELECT
    transaction_type,
    COUNT(*) AS total_transactions,
    SUM(transaction_amount) AS total_revenue
FROM dbo.transaction_raw
GROUP BY transaction_type;
```

### Fraud Transaction Analysis

```sql
SELECT
    fraud_flag,
    COUNT(*) AS total_transactions
FROM dbo.transaction_raw
GROUP BY fraud_flag;
```

## Key Business Insights

- Transfer transactions generated the highest revenue
- Fraud transactions represented approximately 48.10% of all transactions
- Mobile and desktop devices showed different transaction behaviors
- Certain network slices generated higher transaction revenue
- Transaction activity trends can support operational monitoring
